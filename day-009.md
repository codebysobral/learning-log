# Day 009

## Topics Studied
- Git Rebase vs. Git Merge
- Linear history vs. merge history
- Resolving conflicts during rebase
- Golden rule: never rebase shared commits

## What I Learned
I learned that `git rebase` creates a cleaner, linear commit history, while `git merge` preserves the complete branch history. I also practiced resolving conflicts during a rebase and understood why rebasing shared commits should be avoided.

## Practice
- Simple rebase
- Rebase with conflicts (`git rebase --continue`)
- Compare history with `git log --graph`

## Challenges
Understanding when to use rebase instead of merge and resolving conflicts without losing changes.

## Notes
Rebase is great for keeping history clean, but it should only be used on local commits that haven't been shared yet.
