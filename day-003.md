# Day 003

## Topics Studied

* git diff and git diff --staged
* git restore and git restore --staged
* git commit --amend
* .gitignore

## What I Learned

Today I learned how to view changes in my code before they are in the staging area and after they have been staged, using the `git diff` and `git diff --staged` commands, respectively.

I also learned how to restore files after making changes using the `git restore` command, as well as how to remove them from the staging area using the `git restore --staged` command.

I learned how to rename the most recent commit using `git commit --amend -m "new-name"`.

Finally, I learned how to use `.gitignore` and understood its purpose and how it works.

## Practice

* Viewing diffs before and after staging
* Undoing changes at different stages
* Correcting a commit message
* Creating a functional .gitignore

## Challenges

I found it a bit difficult to think of real-world applications for these commands, but as I tested and practiced them, I began to see how useful they are.
