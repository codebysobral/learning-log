# Day 008

## Topics Studied

* `git stash`, `git stash list`, `git stash pop`, `git stash apply`
* `git stash drop` and `git stash clear`
* Descriptive stash messages (`git stash push -m`)

## What I Learned

Today I learned how to temporarily save uncommitted changes using `git stash` without creating a commit. I also learned how to list, apply, recover, and remove stashes. Additionally, I understood the importance of using descriptive messages to identify stashes more easily when working on multiple tasks.

## Practice

* Saved and restored changes using `git stash`
* Worked with multiple stashes at the same time
* Practiced the difference between `git stash pop` and `git stash apply`

## Challenges

* Remembering when to use `pop` instead of `apply`
* Managing multiple stashes without getting confused

## Notes

Using descriptive stash messages makes it much easier to organize temporary work, especially when switching between different tasks or features.
