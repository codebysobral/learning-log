# Day 002

## Topics Studied
- Staging area and the three states of a file in Git
- `git add` (specific file and `git add .`)
- `git commit` and commit messages
- `git log` and `git log --oneline`

## What I Learned
Today I learned about the **Working Directory**, which is where I 
create, edit, and delete files while working on my project.

I also learned about the **Staging Area**, a temporary area where I can 
choose which changes I want to include in my next commit.

Finally, I learned about the **Repository**. After selecting the changes 
in the Staging Area, I can create a commit. That commit becomes part of 
the project's history and can later be pushed to a remote repository 
like GitHub.


## Practice
- Creating and staging files
- First real commit
- Repeating the full add → commit workflow three times

## Challenges
It took me a while to understand the difference between each file state
