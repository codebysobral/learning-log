# Day 015

## Topics Studied
- Special GitHub profile repository
- Structure of a professional profile README
- Essential Markdown (headings, lists, bold text, and links)
- Best practices and common mistakes in profile READMEs

## What I Learned
Today I learned how GitHub profile repositories work and why they're useful for creating a strong first impression. I also became more comfortable with basic Markdown and realized that a good README is more than just a list of skills—it's a way to introduce myself and highlight my journey as a developer.

## Practice
- Created my `codebysobral` profile repository
- Wrote the first version of my profile README
- Published it and checked how it looks on GitHub

## Challenges
At first, I was a little confused about how the special profile repository works and what kind of information should go into the README. It also took me some time to organize the content in a way that looked clean and professional.

## Notes
A simple and well-organized README can have a bigger impact than one that's overloaded with information. I'll keep improving mine as I learn new technologies and build more projects.
